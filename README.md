# ⬡ CELVRA
### *Beyond Storage, Beyond Limits*

> A personal cloud storage platform with Crystal Drive interface — built for privacy, control, and beauty.

---

## 🌐 Live Demo
Deploy to GitHub Pages to access from any device.

---

## ✨ Features

| Feature | Description |
|---|---|
| 🔐 **Secure Auth** | Email/password login with Remember Me, unique email per account |
| 🖼️ **Photos** | Upload, organize, delete images with folder support |
| 🎵 **Music** | Music library with folder organization |
| 🎬 **Videos** | Video file storage |
| 📒 **Contacts** | VCard (.vcf) file imports only |
| 📄 **Documents** | PDF, Word, Excel, TXT files |
| 📱 **APK Files** | Android application packages |
| 💻 **EXE Files** | Windows 32-bit & 64-bit executables |
| 🔗 **Links** | Save and organize important URLs with custom names |
| 📤 **File Transfer** | Send files to other Celvra users by email |
| 🔔 **Notifications** | Real-time notifications with sound when receiving files |
| 📁 **Folders** | Create, rename, delete custom folders per section |
| 📊 **Dashboard** | Overview of all files, stats, and recent activity |

---

## 🚀 Deploy to GitHub Pages

1. Fork or clone this repository
2. Go to **Settings → Pages**
3. Set source to `main` branch, `/ (root)`
4. Your site will be live at `https://yourusername.github.io/Celvra/`

---

## 🔒 Privacy

All data is stored **locally in your browser** using `localStorage`.  
No servers. No tracking. No ads. Your data never leaves your device.

Each user's files are isolated — completely private and independent.

---

## 📋 Supported File Types

- **Photos:** JPG, JPEG, PNG, GIF, BMP, WEBP, SVG
- **Music:** MP3, WAV, FLAC, OGG, AAC, M4A
- **Videos:** MP4, MOV, AVI, MKV, WEBM
- **Contacts:** VCF only
- **Documents:** PDF, DOC, DOCX, XLS, XLSX, TXT, RTF
- **APK:** Android packages
- **EXE:** Windows 32-bit & 64-bit executables

---

## 📤 File Transfer System

- Send files to any registered Celvra user using their email
- Recipient receives a **notification with sound**
- Sender's username is shown (email stays private)
- Recipient must **accept** the file to save it to their library
- Files route automatically to the correct section
